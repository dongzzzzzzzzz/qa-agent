# Services 模块 - Bark 集成（Web SRP）业务规则

> **知识来源**: `senior-qa-brain/scripts/Bark-integration-SRP-web-testcases.md`（生成时间 2026-03-17）  
> **范围**: Gumtree Services SRP（Web），接入 Bark 三方广告后以广告形式展示；埋点不在本批范围。

## 1. 功能概述

- **功能描述**: 在 Services 类目搜索结果页（SRP）展示 Gumtree 与 Bark 来源的服务类广告卡片，支持实验分流、地域/距离/排序筛选、分页浏览，并跳转 H5 落地页。
- **用户角色**: 访客（visitor，默认未登录）；若需登录类用例再单独补充。
- **入口位置**: Gumtree 首页 → Services 类目入口 → `/business-services`（SRP）；Header 可注入实验覆盖。
- **依赖模块**: 搜索与列表服务、广告库存与排序策略、Bark 数据源接入、前端 SRP 与分页组件。

**默认测试环境（可替换子域）**: `https://www.unicorn.gumtree.io/business-services`（`unicorn` 可换 `bixi`、`zoidberg` 等）。

## 2. 核心流程

### 2.1 主流程

1. 设置实验 Header `gt_gb_exp_ovr`（如 `e_002:A` / `e_002:B`）或不传（默认策略）。
2. 从首页或导航进入 Services SRP（`/business-services`）。
3. 使用 Location（地名/邮编）、距离（如 +1 mile / +3 mile）、排序（Most recent first / Nearest first）筛选与翻页。
4. 浏览列表中的广告卡与普通卡：Gumtree 广告优先；当前 query 下 Gumtree 耗尽后展示 Bark。
5. 通过按钮文案识别 Bark（见 3.4）；点击卡片进入 H5 落地页并返回 SRP。

### 2.2 异常流程

| 场景 | 处理方式 |
|------|----------|
| Location 无效邮编或超长/特殊字符 | 明确提示、截断或不触发搜索；不崩溃、不 500 |
| 无结果 query | 展示空结果态；不出现 Gumtree/Bark 广告卡（TC047） |
| 非法页码（URL 手改 0/-1/超大） | 重定向合法页或提示；不 500 |
| 不传或空字符串 `gt_gb_exp_ovr` | 与「不传 header」行为一致（TC054） |
| 未知实验值（如 `e_002:C`） | 页面可用，按默认或回退策略（待产品定义） |

## 3. 业务规则

### 3.1 输入规则

| 字段 | 类型 | 必填 | 长度/格式 | 默认值 | 说明 |
|------|------|------|-----------|--------|------|
| `gt_gb_exp_ovr` | Header 字符串 | 否 | `e_002:A` / `e_002:B` 等 | 无 | 控制新老实验；未知值需降级策略 |
| Location | 地名或邮编 | 否 | 邮编需合法；可 trim/大小写兼容 | 空=未定义区域（全区域） | 修改 Location/距离/排序视为新 query，广告「耗尽」状态重置 |
| 距离 | 枚举 | 视产品 | 如 +1 mile、+3 mile | 产品默认 | 与 Location 联动；全区域下行为需按产品定义 |
| 排序 | 枚举 | 否 | Most recent first / Nearest first 等 | 产品默认 | 切换排序应回到第 1 页并刷新列表 |
| 分页 | 正整数 | 否 | 合法页码 | 1 | 翻页后筛选/排序条件保持 |

### 3.2 校验规则

- **Bark 识别（UI）**: 卡片 CTA 为 **「Request a call」** 或 **「Request a call back」** 即为 Bark；否则为 Gumtree（或普通卡）。
- **卡片展示字段**: 列表卡仅展示 **Location** + **Category**（Bark 可额外含 CTA）；广告卡与普通卡字段一致（TC024、TC029）。
- **广告优先级**: Gumtree 广告优先；当前 query 下 Gumtree 未耗尽前不应出现 Bark；耗尽后出现 Bark；同一 query 下 Bark 出现后 Gumtree 不应「回流」（除非刷新或条件变化）（TC031–TC035、TC046、TC055）。
- **A/B 一致性**: `e_002:A` 与 `e_002:B` 下广告 Gumtree→Bark 规则应一致，除非需求明确允许差异（TC055）。

### 3.3 权限规则

- 默认用例为 **visitor**；无登录账号要求。若后续扩展登录场景，单独补充规则文档。

### 3.4 业务约束

- **实验**: 需通过 Header 注入；A/B 下 SRP 核心能力（Location、距离、排序、翻页）均应可用（TC003）。
- **落地页**: Web 点击卡片进入 **H5**；埋点验收不在本文档范围。
- **分页**: 页码分页；排序切换后回到第 1 页；快速连续翻页/浏览器前进后退行为需符合产品定义。

### 3.5 站点与环境

| 项 | 说明 |
|----|------|
| 站点 | 文档示例为 `uk`，按实际环境填写 |
| 基础 URL | 默认 `https://www.{env}.gumtree.io/business-services` |
| Session/账号 | 文档示例 `relic_web` 用于 session 命名，可按团队规范替换 |

## 4. 错误处理

| 错误/异常类型 | 触发条件 | 用户侧表现 | 备注 |
|----------------|----------|------------|------|
| 无效邮编 | 格式/长度不合法 | 明确错误或不再搜索 | TC007 |
| 超长 Location | >256 字符 | 提示或截断，不崩溃 | TC039 |
| 特殊字符 | `'"<>;` 等 | 无脚本注入、页面异常 | TC040 |
| 非法页码 | URL 篡改 | 合法页或提示 | TC049 |
| 列表/接口错误 | 服务异常 | 非 500 错误页、列表可恢复 | TC020 等稳定性用例 |

> 无统一「业务错误码」表时，以页面提示与 HTTP 状态为准。

## 5. 已知问题

- **默认实验策略**: 不传 `gt_gb_exp_ovr` 时的默认落点需 **产品/研发确认**（TC004）。
- **广告耗尽回退**: 「仅 Bark 或仅 Gumtree」等数据构造依赖环境，部分用例标为不可自动化（TC023）。
- **广告去重**: 跨页重复曝光是否允许依赖产品规则（TC036）。
