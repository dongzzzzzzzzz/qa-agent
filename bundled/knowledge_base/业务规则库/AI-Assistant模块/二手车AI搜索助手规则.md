# 二手车AI搜索助手规则

## 1. 功能概述

### 业务价值
- **降低搜索门槛**: 用户无需了解汽车专业术语,通过自然语言描述需求即可获得精准推荐
- **提升搜索效率**: AI理解用户意图,快速推荐匹配车辆,减少用户筛选时间
- **增强用户体验**: 对话式交互替代传统筛选器,更符合现代用户习惯
- **提高转化率**: 精准推荐提升用户找到心仪车辆的概率,促进成交

### 用户角色
- **主要用户**:
  - 首次购车者: 不了解汽车参数,需要AI指导
  - 忙碌用户: 希望快速找到心仪车辆,节省时间
  - 对话偏好用户: 更喜欢聊天式交互而非表单填写
- **次要用户**:
  - 专业买家: 快速验证AI推荐准确性
  - 价格敏感用户: 通过对话发现隐藏优惠

### 入口位置
- **主入口**: Cars分类页 → "AI Search"入口
- **辅助入口** (可选):
  - 首页搜索框 → "试试AI搜索"提示
  - 车辆列表页 → "AI帮你找车"引导

## 2. 核心流程

### 2.1 主流程

| 步骤 | 操作 | 系统响应 | 观测点 |
|------|------|---------|--------|
| 1 | 用户进入AI Search页面 | 展示欢迎消息、热门标签、输入框 | 页面加载≤2秒 |
| 2 | 用户输入搜索需求(文本/语音) | 发送消息,显示加载状态 | 输入长度≤500字符 |
| 3 | AI处理请求 | 3秒内返回推荐车辆或澄清问题 | AI响应时间P95≤3秒 |
| 4 | 展示推荐车辆卡片 | 横向滑动列表,每屏2.5个卡片 | 推荐数量3-10个 |
| 5 | 用户查看/点击车辆 | 跳转车辆详情页或收藏 | 点击响应≤500ms |
| 6 | 用户提供反馈(可选) | 记录点赞/点踩,可重新生成 | 反馈数据成功上报 |

### 2.2 异常流程

| 场景 | 触发条件 | 系统处理 | 用户体验 |
|------|---------|---------|---------|
| AI理解失败 | 用户输入过于模糊 | 返回澄清问题 | "您的预算范围是多少?" |
| 无匹配车辆 | 搜索条件过于严格 | 提示调整条件或推荐接近车辆 | "暂时没有符合您需求的车辆" |
| 网络异常 | 发送消息时网络中断 | 10秒超时,提示重试 | "网络连接失败,请检查后重试" |
| AI响应超时 | AI处理超过10秒 | 显示超时提示 | "AI处理超时,请稍后重试" |
| 语音识别失败 | 语音转文本失败 | 提示重新录音或切换文本输入 | "语音识别失败,请重试" |
| 麦克风权限被拒 | 用户拒绝麦克风权限 | 显示引导,提供"去设置"入口 | "麦克风权限未开启,无法使用语音搜索" |

## 3. 业务规则

### 3.1 输入规则

| 规则ID | 规则描述 | 约束条件 | 错误提示 |
|--------|---------|---------|---------|
| INPUT-001 | 文本消息长度限制 | 1-500字符 | "消息长度不能超过500字符" |
| INPUT-002 | 空消息禁止发送 | 非空且非全空格 | 发送按钮禁用 |
| INPUT-003 | 语音录音时长限制 | ≤60秒 | "录音时长已达上限(60秒)" |
| INPUT-004 | 快速标签选择上限 | ≤10个标签 | "最多选择10个标签" |
| INPUT-005 | 标签与文本混合输入 | 允许,拼接为一条消息 | - |

### 3.2 校验规则

| 规则ID | 规则描述 | 校验时机 | 校验逻辑 |
|--------|---------|---------|---------|
| VAL-001 | 消息内容非空校验 | 点击发送前 | trim()后长度>0 |
| VAL-002 | AI响应超时校验 | AI处理中 | 超过10秒触发超时 |
| VAL-003 | 车辆数据完整性校验 | AI返回推荐后 | 必须包含标题、价格、图片URL |
| VAL-004 | 语音文件大小校验 | 录音完成后 | 文件大小≤10MB |
| VAL-005 | 会话有效性校验 | 每次请求前 | session_id存在且未过期 |

### 3.3 权限规则

| 规则ID | 规则描述 | 适用角色 | 说明 |
|--------|---------|---------|------|
| PERM-001 | 麦克风权限 | 所有用户 | 首次使用语音功能需请求权限 |
| PERM-002 | 游客模式功能限制 | 游客用户 | 可使用AI搜索,但历史对话不持久化 |
| PERM-003 | 收藏功能权限 | 登录用户 | 游客收藏需先登录 |
| PERM-004 | 历史对话访问权限 | 登录用户 | 仅本人可查看历史对话 |
| PERM-005 | 车辆对比功能 | 所有用户 | 无权限限制 |

### 3.4 业务约束

| 约束ID | 约束描述 | 约束值 | 业务原因 |
|--------|---------|-------|---------|
| CONS-001 | 推荐车辆数量范围 | 3-10个 | 避免选择不足或信息过载 |
| CONS-002 | 车辆对比数量限制 | 2-3个 | 最少2个保证对比意义,最多3个避免眼花缭乱 |
| CONS-003 | 推荐问题固定数量 | 3个 | 引导用户继续探索,但不过多打扰 |
| CONS-004 | 会话历史保留期限 | 30天 | 平衡存储成本和用户体验 |
| CONS-005 | 反馈撤销时间窗口 | 5秒内 | 允许误操作撤销,但不影响数据分析 |
| CONS-006 | AI响应时间SLA | P95≤3秒 | 保证用户体验流畅 |
| CONS-007 | 图片加载时间SLA | P95≤2秒 | 避免用户等待焦虑 |

## 4. 错误处理

### 4.1 错误码定义

| 错误码 | 错误类型 | 触发场景 | HTTP状态码 |
|--------|---------|---------|-----------|
| AI-1001 | AI理解失败 | 用户输入无法解析 | 200 |
| AI-1002 | AI响应超时 | AI处理超过10秒 | 504 |
| AI-1003 | 无匹配车辆 | 搜索条件无结果 | 200 |
| AI-2001 | 网络连接失败 | 请求发送失败 | - |
| AI-2002 | 语音转文本失败 | 语音识别服务异常 | 500 |
| AI-2003 | 车辆数据异常 | 推荐车辆数据不完整 | 500 |
| AI-3001 | 权限被拒绝 | 麦克风权限未授予 | - |
| AI-3002 | 会话过期 | session_id无效或过期 | 401 |
| AI-4001 | 系统繁忙 | 并发请求过多 | 429 |

### 4.2 错误提示文案

| 错误码 | 用户提示文案 | 操作建议 |
|--------|------------|---------|
| AI-1001 | "抱歉,我没太理解您的需求。可以换个方式描述吗?" | 提供澄清问题引导 |
| AI-1002 | "AI处理超时,请稍后重试" | 提供"重新尝试"按钮 |
| AI-1003 | "暂时没有找到符合您需求的车辆,您可以尝试调整搜索条件" | 提供接近条件的推荐 |
| AI-2001 | "网络连接失败,请检查网络后重试" | 提供"重试"按钮 |
| AI-2002 | "语音识别失败,请重新录音" | 提供"重新录音"或"使用文本输入" |
| AI-2003 | "获取车辆信息失败,请稍后重试" | 自动跳过异常车辆,显示其他推荐 |
| AI-3001 | "麦克风权限未开启,无法使用语音搜索" | 提供"去设置"入口 |
| AI-3002 | "会话已过期,请重新开始" | 自动刷新页面 |
| AI-4001 | "系统繁忙,请稍后再试" | 提供"稍后重试"按钮 |

## 5. 依赖模块

### 5.1 上游依赖(谁调用我)

| 模块名称 | 依赖关系 | 调用场景 | 数据传递 |
|---------|---------|---------|---------|
| Cars分类页 | 入口跳转 | 用户点击"AI Search"入口 | 无,直接进入AI Search页面 |
| 首页搜索框(可选) | 功能引导 | 提示"试试AI搜索" | 可传递初始搜索关键词 |
| 车辆列表页(可选) | 功能引导 | 筛选无结果时提示"试试AI搜索" | 可传递当前筛选条件 |

### 5.2 下游依赖(我调用谁)

| 模块名称 | 依赖关系 | 调用场景 | 数据传递 |
|---------|---------|---------|---------|
| **搜索业务域** | 数据流转(核心) | AI推荐车辆时调用车辆搜索数据 | 搜索条件(价格、里程、车型等) |
| 车辆详情页 | 功能跳转 | 用户点击车辆卡片 | 车辆ID (vehicle_id) |
| 收藏系统 | 功能调用 | 用户点击收藏按钮 | 车辆ID + 用户ID |
| Messages业务域 | 功能跳转(可选) | 用户联系卖家 | 车辆ID + 卖家ID |
| AI推荐服务(后端) | API调用 | 每次用户发送消息 | 消息文本 + session_id + 用户画像 |
| 语音识别服务(后端) | API调用 | 语音录音完成后 | 音频文件 |

### 5.3 跨域交互说明

#### 与搜索业务域的交互
- **数据流转**: AI-Assistant业务域通过API调用搜索业务域,获取匹配的车辆列表
- **数据格式**: 
  ```json
  {
    "filters": {
      "price_max": 5000,
      "fuel_type": "Petrol",
      "mileage_max": 50000
    },
    "limit": 10
  }
  ```
- **依赖风险**: 如果搜索服务不可用,AI Search将无法返回推荐车辆
- **降级策略**: 显示"暂时无法获取车辆信息,请稍后重试",建议用户使用传统搜索

#### 与Messages业务域的交互
- **功能依赖**: 用户从AI推荐的车辆卡片点击"联系卖家"时,跳转到Messages页面
- **数据传递**: 车辆ID + 卖家ID → Messages页面自动创建对话
- **依赖风险**: 如果Messages功能不可用,用户无法联系卖家
- **降级策略**: 显示"消息功能暂时不可用",提供卖家电话(如果有)

#### 与收藏系统的交互
- **功能依赖**: AI推荐的车辆收藏状态需与现有收藏系统同步
- **数据一致性**: 用户在AI Search中收藏的车辆,必须在"My Favourites"页面可见
- **依赖风险**: 如果收藏接口失败,用户会困惑为何收藏不成功
- **降级策略**: 显示"收藏失败,请稍后重试",同时缓存收藏操作,网络恢复后重试

### 6.2 技术风险

| 风险ID | 风险描述 | 影响 | 缓解措施 |
|--------|---------|------|---------|
| RISK-001 | AI理解错误率高 | 推荐不相关车辆,用户流失 | 边界词汇测试,持续优化AI模型 |
| RISK-002 | AI响应时间过长(>5秒) | 用户等待焦虑,放弃使用 | 设置超时机制,优化AI服务性能 |

| RISK-005 | 语音识别准确率低(噪音环境) | 用户体验下降 | 提供文本预览编辑功能 |
| RISK-006 | 网络异常导致消息发送失败 | 用户困惑,可能重复发送 | 本地缓存,网络恢复后自动重试 |
| RISK-007 | 并发请求过多导致服务降级 | 部分用户无法使用 | 实施限流策略,提示"系统繁忙" |
