# 帮助中心规则

## 1. 功能概述
- **功能描述**: Gumtree Help Desk 帮助中心是为用户提供自助服务的知识库平台,包含搜索功能、导航树、文章详情展示、联系功能等核心模块
- **用户角色**: 访客(无需登录)
- **入口位置**: https://help.gumtree.com/s/ (帮助中心首页)
- **依赖模块**: Live Chat 第三方客服系统、OneTrust Cookie 合规模块

## 2. 核心流程

### 2.1 主流程
1. 用户访问帮助中心首页,查看 Top FAQ 和导航树
2. 用户可通过搜索框输入关键词查找帮助文章
3. 用户可通过左侧导航树浏览分类和主题
4. 用户点击文章标题进入详情页查看完整内容
5. 用户可通过 Contact Us 页面提交咨询或使用 Live Chat

### 2.2 异常流程
- 搜索关键词不存在 → 显示 "0 results" 页面,建议提供后续引导
- 空搜索提交 → 显示必填提示或跳转到 0 结果页
- 特殊字符/超长字符串搜索 → 正确编码或过滤,不应出现系统错误
- 访问不存在的文章 → 显示友好的 404 错误页面

## 3. 业务规则

### 3.1 输入规则

| 字段 | 类型 | 必填 | 格式 | 说明 |
|-----|------|------|------|------|
| 搜索关键词 | String | 否 | 任意文本 | 页面顶部搜索框,建议100字符以内 |
| Contact Us 主题 | Select | 是 | 预设选项列表 | 联系我们页面的主题选择下拉框 |

### 3.2 校验规则

| 场景 | 规则 | 错误提示 |
|-----|------|---------|
| 搜索框空搜索 | 可能阻止提交或跳转到 0 结果页 | 显示必填提示或 "0 results" |
| 特殊字符搜索 | 特殊字符被正确编码或过滤 | 正常提交,不出现系统错误 |
| 超长字符串搜索 | 建议有字符长度限制(100字符) | 不应导致页面卡顿或错误 |
| SQL注入测试 | 输入被正确过滤或转义 | 不应返回异常数据或数据库错误 |
| XSS攻击测试 | 脚本被过滤或转义,不被执行 | 不应弹出 alert 或执行脚本 |

### 3.3 权限规则

| 角色 | 操作 | 行为 |
|-----|------|------|
| 访客(未登录) | 浏览帮助中心 | 无需登录,可自由访问所有帮助文章 |
| 访客(未登录) | 搜索帮助文章 | 无需登录,可正常使用搜索功能 |
| 访客(未登录) | 查看文章详情 | 无需登录,可查看完整文章内容 |
| 访客(未登录) | 提交反馈 | 可点击 "Yes/No" 反馈按钮 |
| 访客(未登录) | 使用 Contact Us | 可选择主题并查看联系方式 |
| 访客(未登录) | 使用 Live Chat | 可点击 Live Chat 按钮(可能受营业时间限制) |

### 3.4 业务约束

- **搜索结果展示**:搜索结果页 URL 包含 `keysearch=<关键词>`,显示相关文章的标题和简介
- **导航树交互**:
  - 支持展开/折叠分类
  - 可同时展开多个分类(或新展开自动折叠旧分类,取决于设计)
  - 点击子主题跳转到对应文章详情页,并在左侧导航树中高亮显示
- **文章详情页平台切换**:部分文章支持 Web/iOS/Android 平台切换,切换后显示对应平台的指南内容
- **相关文章推荐**:文章底部显示至少3篇相关文章推荐
- **反馈功能**:"Was this article helpful?" 区域提供 Yes/No 按钮,点击后提交反馈数据
- **Live Chat 按钮**:显示在页面右下角,文案 "Hello, have a question? Let's chat.",点击后打开客服窗口或提示营业时间
- **外部链接安全**:外部链接应包含 `rel="noopener noreferrer"` 属性(防止钓鱼攻击)

### 3.5 站点规则

#### UK 站
- 帮助中心 URL: `https://help.gumtree.com/s/`
- Contact Us URL: `https://help.gumtree.com/s/contact-us`
- 搜索结果 URL 格式: `https://help.gumtree.com/s/keysearch?keysearch=<关键词>`
- 文章详情 URL 格式: `https://help.gumtree.com/s/article?article=<文章标识>`

## 4. 错误处理

| 错误场景 | 触发条件 | 用户提示 | 技术处理 |
|---------|---------|---------|---------|
| 搜索结果为0 | 输入不存在的关键词 | 显示 "0 results: <关键词>" | 建议提供相关分类推荐或联系客服入口 |
| 空搜索 | 不输入内容点击搜索 | 显示必填提示或跳转到 0 结果页 | 不应出现页面错误 |
| SQL注入尝试 | 输入 `' OR '1'='1` 等注入字符串 | 搜索结果为0或正常匹配结果 | 输入被正确过滤或转义,不返回异常数据 |
| XSS攻击尝试 | 输入 `<script>alert('XSS')</script>` | 搜索结果正常显示(可能为0结果) | 脚本被过滤或转义,不被执行 |
| URL参数篡改 | 手动修改 URL 参数(如注入脚本) | 页面正常加载,不执行恶意脚本 | 参数值被正确处理(过滤或转义) |
| 访问不存在的文章 | URL 中 article 参数值不存在 | 显示友好的 404 错误页面或"文章不存在"提示 | 不应显示系统错误信息或堆栈跟踪 |

## 5. 依赖模块

### 5.1 上游依赖(谁调用我)
- Gumtree 主站各页面的 "Help & Contact" 链接
- 用户在遇到问题时主动访问帮助中心 URL

### 5.2 下游依赖(我调用谁)
- **Live Chat 第三方客服系统**:通过 Live Chat 按钮调用,提供实时客服支持
- **OneTrust Cookie 合规模块**:可能在首次访问时显示 Cookie 横幅

### 5.3 跨域交互说明
- 帮助中心是独立的子域(help.gumtree.com),与主站(www.gumtree.com)数据不直接共享
- Live Chat 功能依赖第三方客服系统,可能受营业时间和客服在线状态影响

## 6. 已知问题

### 6.1 产品待确认问题
- **TC003**:空搜索的具体行为(是阻止提交还是跳转到 0 结果页)需明确 ⚠️
- **TC010**:导航树展开状态在页面刷新后是否保持,需明确产品设计 ⚠️
- **TC019**:Live Chat 点击后的具体行为(打开窗口 vs 跳转页面 vs 提示营业时间)需明确 ⚠️
- **TC024**:Safari 浏览器兼容性需在真实环境中补充测试 ⚠️
- **TC025**:页面加载性能指标(首页加载时间 < 3秒)需明确并纳入监控 ⚠️

### 6.2 技术风险
- Live Chat 功能依赖第三方系统,若第三方服务不可用,用户无法使用实时客服
- 搜索功能若未正确过滤恶意输入,可能存在 SQL 注入或 XSS 攻击风险
- 外部链接若未添加 `rel="noopener noreferrer"` 属性,可能存在钓鱼攻击风险

### 6.3 测试过程中发现的问题
- 无(本文档基于测试用例归档,实际测试执行后需补充)
