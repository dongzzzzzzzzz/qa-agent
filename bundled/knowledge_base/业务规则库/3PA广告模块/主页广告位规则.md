# 3PA广告模块 - 主页广告位业务规则

> **文档版本**: v1.0
> **最后更新**: 2026-04-16
> **维护团队**: Web QA Team
> **关联需求**: Gumtree 主页第三方广告位（3PA - Third Party Ads）

---

## 1. 功能概述

### 1.1 功能描述
第三方广告位（3PA）是 Gumtree 主页的核心收入模块，通过在页面预留广告容器，异步加载第三方广告内容（Google Ads 等），在不影响主内容展示的前提下实现广告变现。主页包含 4 个广告位容器，支持横幅广告、侧边栏广告和追踪像素。

### 1.2 用户角色
- **访客/用户（Visitor/User）** - 浏览主页时被动接收广告展示
- **广告系统（Ad System）** - Google Ads 等第三方广告 SDK，负责向广告位注入内容

### 1.3 入口位置
- **主页**：`${BASE_URL}`（根据环境配置选择对应域名）
- 广告位在页面加载时自动创建，无需用户主动触发

### 1.4 依赖模块
- **Google Ads SDK** - 通过 iframe 注入广告内容
- **Cookie 隐私协议** - 用户必须接受 Cookie 协议后广告才能加载
- **Sentry 监控** - 第三方错误监控脚本

---

## 2. 核心流程

### 2.1 主流程：广告位加载与展示

1. 用户访问 Gumtree 主页
2. 页面 DOM 加载，4 个广告位容器预创建（含 `ad-slot` class 和 `data-display-ad="true"` 属性）
3. Cookie 隐私弹窗出现，用户点击 "Accept All" 接受协议
4. 广告脚本异步加载（不阻塞主内容）
5. Google Ads SDK 向广告位容器内注入 iframe
6. iframe 加载广告创意内容（图片/富媒体）
7. 广告展示完成，追踪像素触发曝光统计

### 2.2 异常流程

- **用户未接受 Cookie 协议**：广告脚本不加载，广告位容器存在但无内容（尺寸为 0）
- **广告未投放**：容器存在但尺寸为 0，不影响页面布局
- **广告脚本加载失败**：容器为空，主内容不受影响，控制台可能出现 ERR_FAILED 错误
- **广告被拦截器屏蔽**：容器可能被移除或隐藏，页面主内容正常展示
- **网络超时**：广告加载超过 10 秒自动放弃，核心功能不受影响

## 3. 业务规则

### 3.1 广告位配置规则

#### 主页广告位清单

| 广告位名称 | 容器ID | 别名 | 类型 | 子元素ID | 说明 |
|-----------|--------|------|------|---------|------|
| 顶部广告位 | `top_takeover-container` | top | 横幅广告 | `top_takeover` | 页面顶部导航栏下方，全屏宽度 |
| 右侧广告位 | `homeSideAd-container` | topRight | 侧边栏广告 | `homeSideAd` | 页面右侧，桌面端可见 |
| 备用横幅 | `homeBanner-container` | - | 备用横幅 | `homeBanner` | 当前未启用，特定活动时可能启用 |
| 追踪像素 | `pixel-container` | - | 追踪像素 | `pixel` | 不占用可视空间，用于统计追踪 |

#### 广告位 DOM 结构规则

| 规则项 | 说明 |
|--------|------|
| CSS 类名 | 所有广告位容器必须包含 `ad-slot` class |
| 数据属性 | 所有广告位必须包含 `data-display-ad="true"` 属性 |
| ID 唯一性 | 所有广告位 ID 必须唯一，不可重复 |
| 嵌套结构 | 外层容器（`*-container`）用于布局，内层 div（无 `-container` 后缀）用于广告注入 |
| 容器预留 | 页面加载时即创建容器，不依赖广告是否成功投放 |

### 3.2 广告位尺寸规则

#### 顶部广告位（top_takeover）

| 属性 | 值 | 说明 |
|------|------|------|
| 位置 | top=106px, left=0 | 导航栏下方，左对齐 |
| 容器宽度 | 视口宽度（如 1523px） | 全屏宽度 |
| 容器高度 | 271px | 固定高度，不随内容变化 |
| iframe 标准尺寸 | 970x250px | IAB Large Leaderboard |
| iframe ID 格式 | `google_ads_iframe_/5144/desktop/home/top_0` | Google Ads 命名规范 |

#### 右侧广告位（homeSideAd）

| 属性 | 值 | 说明 |
|------|------|------|
| 位置 | x≈1120px, y≈393px | 主内容区右侧 |
| 标准尺寸 | 300x250 / 300x600 / 336x280 | IAB Medium Rectangle / Half Page / Large Rectangle |
| 可见性条件 | 视口宽度 > 1024px | 桌面端显示 |
| 未投放状态 | 容器存在，尺寸为 0 | 不影响页面布局 |

#### 追踪像素（pixel）

| 属性 | 值 | 说明 |
|------|------|------|
| 宽度 | 1350px | 主内容宽度 |
| 高度 | 0 | 不占用可视空间 |
| 用途 | 嵌入 1x1 透明图片或脚本 | 统计和追踪 |

### 3.3 广告位状态枚举规则

| 状态 | 容器存在 | 尺寸 | 内容 | 说明 |
|------|---------|------|------|------|
| 已投放已加载 | ✅ | 有尺寸（>0） | iframe/图片正常展示 | 正常状态 |
| 已预留未投放 | ✅ | 尺寸为 0 | 空的子元素 | 广告未投放的正常状态 |
| 加载失败 | ✅ | 尺寸为 0 | 容器为空 | 广告脚本加载失败 |
| 被屏蔽 | ❌/✅ | 不确定 | 被移除或隐藏 | 广告拦截器干预 |

### 3.4 Cookie 隐私前置规则

| 规则项 | 说明 |
|--------|------|
| 前置条件 | 用户必须点击 "Accept All" 接受 Cookie 隐私协议 |
| 未接受时行为 | 广告脚本不加载，广告位容器存在但无内容 |
| 影响范围 | 模块 8 所有用例（TC-3PA-021 ~ TC-3PA-026） |
| 弹窗出现环境 | Production 和 Staging 必定出现；测试环境可能不出现 |

### 3.5 性能约束规则

| 约束项 | 标准值 | 说明 |
|--------|--------|------|
| 页面加载时间（含 Cookie 操作） | < 5 秒 | 主内容在 2 秒内展示完成 |
| 广告加载超时 | < 10 秒 | 超时后放弃加载 |
| 加载方式 | 异步（async/defer） | 不阻塞主内容展示 |
| 广告失败影响 | 无影响 | 广告加载失败不影响搜索、浏览、导航等核心功能 |
| DOMContentLoaded | < 2 秒 | 主内容展示时间 |

### 3.6 响应式布局规则

| 视口类型 | 宽度 | 顶部广告位 | 右侧广告位 | 说明 |
|---------|------|-----------|-----------|------|
| 桌面端 | 1523x800 | ✅ 展示，全屏宽度 | ✅ 展示（投放时） | 标准测试视口 |
| 平板端 | 768x1024 | ✅ 展示，尺寸适配 | ⚠️ 可能隐藏或调整位置 | iPad |
| 移动端 | 375x667 | ⚠️ 可能调整尺寸或隐藏 | ❌ 隐藏或移至底部 | iPhone SE |

**环境差异关键点**：
- 所有环境的广告位容器结构应保持一致
- 测试环境可能没有实际广告投放，广告内容验证用例会自动跳过
- Staging 最接近生产环境，建议在 Staging 完成完整测试后再部署到 Production

## 4. 错误处理

| 错误场景 | 触发条件 | 系统行为 | 用户影响 |
|---------|---------|---------|---------|
| 广告脚本加载失败 | 网络请求失败（CORS、404） | 控制台输出 ERR_FAILED 错误 | 无影响，主内容正常展示 |
| 第三方监控脚本错误 | Sentry.js TypeError | 控制台输出 TypeError | 无影响 |
| 广告投放为空 | 广告位无预算/时段外 | 容器存在但尺寸为 0 | 无影响，不出现大片空白 |
| 广告被拦截 | 用户安装 AdBlock/uBlock | 容器可能被移除或隐藏 | 核心功能不受影响 |
| Cookie 未接受 | 用户未点击 Accept All | 广告脚本不加载 | 广告不展示，页面正常 |
| iframe 加载超时 | 广告 CDN 响应慢 | 超过 10 秒放弃加载 | 无影响 |

### 5.2 技术风险

| 风险编号 | 风险描述 | 缓解措施 | 状态 |
|---------|---------|---------|------|
| R1 | 广告投放不稳定，可能因地域/时间/预算等原因不投放 | 广告内容用例设计为智能跳过（pytest.skip） | ✅ 已处理 |
| R2 | 第三方广告 SDK 依赖外部服务，可能因服务故障失败 | 广告失败不影响核心功能，防御性设计 | ✅ 已验证 |
| R3 | 不同浏览器的广告展示可能有差异 | 标准化使用 Chrome/Chromium (Playwright) 测试 | ⚠️ 需扩展浏览器覆盖 |
| R4 | Cookie 隐私法规变化可能影响广告投放机制 | 关注 GDPR/UK-GDPR 法规变化 | ⚠️ 持续关注 |
