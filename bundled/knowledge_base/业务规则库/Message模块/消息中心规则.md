# Message模块 - 消息中心业务规则

## 1. 功能概述

- **功能描述**: Gumtree 消息中心（Messages），为买卖双方提供会话管理、实时消息沟通、媒体分享、议价出价、举报/拉黑等全套通信功能
- **用户角色**: Seller（广告发布者，被动接收，主动回复）、Buyer（广告浏览者，主动发起会话）
- **入口位置**:
  - 顶部导航 "Messages" 链接 → `/manage/messages`
  - 广告详情页（VIP）"Contact Seller" / "Message" 按钮（Cars类目提供消息输入框；For Sale类目直接跳转消息中心）
- **依赖模块**: 广告模块（广告状态同步影响功能可用性）、用户认证模块（未登录拦截）、Make an Offer议价功能（与广告状态强绑定）

---

## 2. 核心流程

### 2.1 主流程（Buyer 发起会话 → Seller 回复）

1. Buyer 在广告详情页点击 "Contact Seller" / "Message" 发起首条消息
2. 系统创建会话，Buyer 跳转至消息中心，会话出现在列表顶部
3. Seller 登录消息中心，在会话列表中看到来自 Buyer 的新会话
4. Seller 打开会话，查看 Buyer 消息（左侧气泡），输入回复消息
5. Seller 点击发送（或按 Enter），消息显示在右侧绿色气泡，Buyer 侧会话列表预览更新
6. Buyer 打开会话，查看 Seller 回复

### 2.2 异常流程

- **未登录访问** `/manage/messages`：重定向到登录页或弹出登录弹窗，不允许直接访问
- **未登录 Buyer 点击 "Contact Seller"**：触发登录流程（弹出登录对话框或跳转登录页）
- **越权访问他人 conversationId**：不展示会话内容，跳转权限错误页或消息中心首页
- **广告已删除/过期状态**：文字/图片/视频消息仍可正常发送；Make an Offer 功能不可用
- **重复联系同一广告**：不创建新会话，直接跳转已有会话

## 3. 业务规则

### 3.1 输入规则

| 字段 | 类型 | 必填 | 格式/限制 | 说明 |
|-----|------|-----|----------|------|
| 文字消息 | String | 否（有附件时可为空） | 任意文本，不能为空/纯空格 | 支持超长消息（至少1000字符） |
| 图片附件 | File | 否 | image/*；单次最多5张；单张建议≤10MB | 超10MB时显示文件过大错误提示或自动压缩 |
| 视频附件 | File | 否 | video/*；单文件≤5MB | 支持多个视频同时上传 |
| 出价金额（自定义输入） | Number | 是（Motors类目） | 数字；≥原价60%；≤原价 | 低于60%或超过原价时前端实时校验拦截 |
| 举报原因 | Enum | 是（举报时） | 8选1 | Fraud/Scam、Abuse/Harassment、Adult/Inappropriate、Phishing Attempt、Spam、Policy Violation、No Reply、No show |

### 3.2 校验规则

**消息发送**
- 输入框为空或纯空格时，发送按钮保持 **disabled**（灰色不可点击）
- 有文字内容或已上传媒体文件时，发送按钮激活变为**绿色可点击**
- Enter 键发送消息（非换行）✅ 已确认（Q3）

**图片上传**
- 单次最多上传 **5 张**图片；尝试上传10张时，系统弹出 alert 提示（消息含 "5" 和 "image"），或缩略图数量被限制在 ≤5 张
- 支持逐张上传（多次调用 file chooser 累积）
- 支持一次性多选（需 input 具备 `multiple` 属性）
- 图片 + 文字可混合发送

**视频上传**
- 单文件大小限制 **≤5MB**
- 支持多个视频同时上传
- 视频 + 文字可混合发送

**出价金额（Make an Offer）**
- 最低出价 = 原价 × 60%（即最大折扣40% off）
- 低于最低限额时：输入框边框变红，显示错误文案 `Your offer needs to be at least £X.XX (40% off)`，提交按钮变灰 disabled
- 恰好等于最低限额：视为合法，提交按钮可用
- 超过原价：不允许提交
- 非数字字符：输入框拒绝录入或显示格式错误提示

### 3.3 权限规则

| 角色 | 发起新会话 | 发送消息（文字/图片/视频） | Make an Offer（VIP页） | Make an Offer（会话框） | Delete/Block/Report |
|------|-----------|--------------------------|----------------------|------------------------|---------------------|
| **Seller** | ❌ 被动接收 | ✅ | ❌ 不显示（查看自己广告时） | ✅（广告活跃时） | ✅ |
| **Buyer** | ✅ 通过广告详情页 | ✅ | ✅（满足正向条件时） | ✅（广告活跃时） | ✅ |
| **未登录** | ❌ 触发登录 | ❌ | ❌ | ❌ | ❌ |

**Make an Offer VIP页正向显示条件**（同时满足以下全部）：
1. Buyer 账号为**非 Pro**（非 Gumtree Pro 订阅用户）
2. Seller 具备 **reply email**
3. 广告属于 **Motors 类目树**
4. 广告类目**非** Cars Wanted（类目 ID **≠ 10301**）
5. 广告状态为**活跃**

**Seller 查看自己广告时不显示 Make offer 入口**（TC025b 已确认）✅

### 3.4 业务约束

**会话相关**
- 同一 Buyer 对同一广告只能有**一条会话**，重复点击 "Contact Seller" 跳转已有会话（不重复创建）
- 会话数据不可跨账号访问（越权访问需拦截）
- conversationId 写入 URL（`/manage/messages?conversationId={id}`），支持刷新保持和浏览器历史导航
- 消息气泡方向：己方消息显示在右侧绿色气泡，对方消息显示在左侧

**会话列表**
- 按最后消息时间**倒序**排列，最近活跃会话在最上方
- 第一页默认最多显示 **30 条**会话
- 左侧会话列表为**独立滚动容器**，与页面主滚动区互相独立
- 浏览器整体滚动（`window.scrollTo`）**不会**触发会话列表翻页
- 只有滚动**左侧列表容器本身**才触发懒加载，加载下一批会话
- 列表头显示数字随懒加载实时更新（"X Conversations" 动态增加）

**未读消息三端一致性**
- 列表头未读数 = 顶栏 Messages 徽章未读数 = 所有可见会话行未读徽章之和 ✅ 已确认（2026-04-07）
- 打开含未读消息的会话后，该会话未读数归零，列表头和顶栏总未读同步减少

**广告状态联动**
- `AD DELETED` / `AD EXPIRED`：会话列表显示角标，文字消息可继续发送，Make an Offer 功能关闭
- `AD DELETED` / `AD EXPIRED` 状态下，广告信息卡仍在聊天面板中显示（保留历史上下文）

**拉黑（Block）**
- 拉黑为**单向操作**，只阻断被拉黑方向主动拉黑方发送消息
- 被拉黑方**不收到任何通知**（静默处理）
- 被拉黑方前端显示发送成功，但消息实际不送达
- 会话历史记录对被拉黑方仍然可见

**删除会话（Delete）**
- 删除为**单边删除**，仅从己方列表移除，对方不受影响
- 删除操作前弹出确认弹窗（防止误操作）
- 删除后即使对方发送新消息，主动删除方的列表中该会话**不会恢复**

**Make an Offer 约束**
- Make an Offer（出价）仅在关联广告**活跃**时可用；广告下线后出价功能同步关闭
- 会话框内为**双向**（买卖双方均可发起）；VIP 页仅 Buyer 可见入口
- For Sale 类目：预设折扣选项（10%, 20%, 30% 等），表单内联展开，不弹新窗口
- Motors 类目：自定义金额输入框（`£ Custom offer`）
- 出价被接受/拒绝后，双方会话均显示结果状态，操作按钮变为不可用（防重复）
- 出价仅对当条会话有效，不影响广告原价

**评价提示（Buyer 专属）**
- 同一会话中 Buyer 累计发送 **5 条消息**后，系统弹出评价提示
- 弹窗包含 "Rate experience if you've bought" 文案，及 "Haven't bought" / "Leave a review" 按钮
- Seller 视角不触发评价提示

**Buyer 发起消息的类目差异**

| 类目 | 广告详情页交互 | 点击后行为 | 目标页面 |
|------|--------------|-----------|---------|
| Cars & Vehicles | "Is this available?" 输入框 + "Send Message" 按钮 | 填写消息后点击发送 | 跳转 `/reply-confirmation/` 确认页 |
| For Sale | 仅 "Message" 按钮（无输入框） | 直接点击跳转 | 跳转 `/manage/messages?conversationId=xxx&openChat=true` |

## 4. 错误处理

| 错误场景 | 错误信息/行为 | 触发条件 | 用户提示 |
|---------|-------------|---------|---------|
| 出价低于原价60% | 输入框边框变红 | 自定义金额 < 原价×60% | `Your offer needs to be at least £X.XX (40% off)` |
| 图片数量超限 | 系统弹出 JavaScript alert | 一次上传超过5张图片 | Alert 消息含 "5" 和 "image" |
| Report 未选原因提交 | Submit 按钮仍可点击，点击后弹窗无任何反馈 | 未选举报原因直接点 Submit | ⚠️ 已知缺陷 BUG-001（应为 disabled 或显示必填提示） |
| 举报成功确认 | 弹窗显示成功信息 | 首次举报且已选原因提交 | 标题：`Thanks. The conversation has been reported to Gumtree`；正文：`Your feedback helps us build a safer community for buyers and sellers.` |
| 重复举报确认 | 弹窗显示已举报状态 | 对同一用户再次举报 | 标题：`Conversation reported`；正文：`Your conversation has been submitted to Gumtree for review.` |
| 未登录访问消息中心 | 重定向/弹窗 | 未登录用户访问 `/manage/messages` | 跳转登录页或弹出登录对话框 |
| 越权访问 conversationId | 跳转错误页或消息中心首页 | 访问他人会话 ID | 不展示会话内容 |

## 5. 已知问题

- **BUG-001**（P1）：Report 弹窗中未选择举报原因时，`Submit` 按钮仍可点击；点击后弹窗保持打开，无任何错误提示或反馈。对应用例 TC022，已标记 `@pytest.mark.known_bug`。建议修复：未选举报原因时 Submit 应为 disabled，或点击后显示必填校验提示。
- **BUG-002**（P2，待环境确认）：消息邮件通知在 unicorn staging 环境中未正常触发（检测窗口10-20分钟内未到达）。对应用例 TC-NOTIFY-01、TC-NOTIFY-02。建议排查：确认 unicorn staging 环境 SMTP/SES 配置状态，检查测试账号是否被屏蔽通知。
