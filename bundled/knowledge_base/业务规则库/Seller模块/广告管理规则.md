# Seller模块 - 广告管理业务规则

## 1. 功能概述
- **功能描述**: 卖家通过Manage Ads功能查看、编辑、删除、推广已发布的广告,支持Active/Expired/Sold等多状态管理
- **用户角色**: 卖家(Seller)
- **入口位置**: 
  - 首页Menu → "Manage my Ads"
  - 发布成功页"Manage my ads"链接
  - 导航栏直接访问`/manage/ads`
- **依赖模块**: 
  - 用户认证模块(登录Session)
  - 广告发布模块(编辑时复用发布表单)
  - 推广模块(推广已发布广告)

## 2. 核心流程

### 2.1 主流程

**查看广告流程**:
1. 用户导航到Manage Ads页面
2. 点击Active ads标签加载广告列表
3. 系统展示广告卡片(标题/价格/发布时间/状态)
4. 鼠标悬停到广告卡片,显示"三个点"操作按钮
5. 点击"三个点" → 展开操作菜单(View ad/Edit ad/Delete ad/Promote ad)
6. 选择"View ad" → 跳转到广告详情页

**编辑广告流程**:
1. 在Manage Ads页面选择广告
2. 点击"三个点" → 点击"Edit ad"
3. 跳转到编辑页面(预填充原广告内容)
4. 页面顶部显示"Update my ad"标题
5. 用户修改字段内容
6. 滚动到底部,点击"Update my ad"按钮
7. 显示确认页面:"Want to move your ad back to the top"(推广推荐)
8. 再次点击"Update my ad"确认提交
9. 成功页面显示"Post another ad"和"Manage my ads"链接

**删除广告流程**:
1. 在Manage Ads页面选择广告
2. 点击"三个点" → 点击"Delete ad"
3. 弹出确认对话框:"Are you sure you want to delete this ad?"
4. 对话框显示"Yes/No"单选按钮
5. 用户选择"Yes"
6. 点击对话框底部"Delete"按钮确认
7. 广告从列表移除,显示成功提示:"Ad deleted successfully"(Toast或页面提示)

**推广广告流程**:
1. 在Manage Ads页面选择广告
2. **直接点击广告卡片上的绿色"Promote"按钮**(不是三个点菜单)
3. 跳转到推广套餐选择页面
4. 页面展示3个推广选项(Featured/Urgent/Spotlight)
5. 勾选推广套餐(可多选)
6. 点击"Continue"按钮
7. 可能出现自动续费确认弹窗:"Please confirm"/"automatically renew"
8. 点击"Confirm"确认(若弹窗出现)
9. 跳转到支付页面
10. 处理支付页面隐私弹窗:"We Care About Your Privacy" → 点击"Accept all"
11. 点击"Pay Now"按钮
12. 触发3D Secure OTP验证(可能):
    - 等待OTP iframe加载(10秒)
    - 在iframe中输入OTP验证码:"1234"(测试环境固定)
    - 点击"SUBMIT"按钮提交
13. 支付成功,跳转到感谢页面(URL包含"thankyou")
14. 页面显示"Post another ad"和"Manage my ads"链接

### 2.2 异常流程

- **广告列表为空**: 显示"No active ads"空状态提示 → 提供"Post an ad"按钮引导发布
- **广告加载失败**: API返回错误 → Toast提示"Failed to load ads. Please refresh." → 显示"Refresh"按钮
- **编辑Session过期**: 编辑过程中Session失效 → 自动跳转登录页 → 登录后返回编辑页(数据保留)
- **删除操作后悔**: 点击Delete后立即关闭弹窗 → 选择"No" → 取消删除,广告保留
- **推广支付失败**: 3D Secure验证失败/支付超时 → 返回推广选择页 → 用户可重新选择或取消
- **OTP验证未触发**: 部分支付不触发3D Secure → 直接跳转感谢页 → 正常流程
- **Promote按钮不可见**: 广告已在推广期内 → "Promote"按钮变灰或隐藏 → 悬停提示"Ad is already promoted"

## 3. 业务规则

### 3.1 广告状态规则

| 状态 | 英文标识 | 说明 | 可执行操作 | 显示位置 |
|-----|---------|------|-----------|---------|
| 在线 | Active | 广告正常展示中 | View/Edit/Delete/Promote | Active ads标签 |
| 已过期 | Expired | 广告超过有效期(30天) | Renew/Delete | Expired ads标签 |
| 已售出 | Sold | 卖家手动标记为已售 | Delete | Sold ads标签(可能) |
| 草稿 | Draft | 未完成发布的草稿 | Continue editing/Delete | Drafts标签(可能) |
| 审核中 | Pending | 后台审核中,未上线 | View(审核状态)/Cancel | Pending ads标签(可能) |
| 已拒绝 | Rejected | 违反规则被拒绝 | View reason/Appeal/Delete | Rejected ads标签(可能) |

**状态转换规则**:
- Active → Expired: 30天后自动过期
- Active → Sold: 用户手动标记(需确认)
- Active → Deleted: 用户删除操作
- Draft → Active: 完成发布流程
- Pending → Active: 审核通过(自动)
- Pending → Rejected: 审核拒绝(自动)

### 3.2 广告列表规则

**列表加载**:
- 默认标签:Active ads
- 排序规则:发布时间倒序(最新在最前)
- 分页规则:每页显示20条(推测) + "Load more"按钮 OR 无限滚动
- 推广卡片过滤:自动过滤"Job Ads"等推广卡片,仅显示真实用户广告

**广告卡片信息**:
- **必显信息**:
  - 广告标题(Title)
  - 价格(Price)
  - 发布时间(如"2 days ago")
  - 主图缩略图(Photos第一张)
- **可选信息**:
  - 推广状态标签(Featured/Urgent/Spotlight)
  - 浏览量(Views count,可能)
  - 收藏量(Favorites count,可能)
  - 回复量(Replies count,可能)
- **操作按钮**:
  - "三个点"菜单(⋮)  - 右侧固定位置
  - "Promote"按钮 - 广告卡片上绿色醒目按钮(仅Active且未推广的广告显示)

**悬停交互**:
- 鼠标悬停到广告卡片 → "三个点"按钮高亮/显示
- 点击"三个点" → 下拉菜单展开(动画:fade-in + slide-down,200ms)
- 菜单选项:View ad / Edit ad / Delete ad (Promote ad可能在三个点菜单中,取决于UI设计)

### 3.3 编辑操作规则

**可编辑字段**:
- ✅ 允许编辑:Photos(增删改) / Title / Description / Price / Item Specifics / Contact / Location
- ❌ 不可编辑:Category(一旦发布不可修改,推测)
- 部分站点可能限制编辑次数(如每月最多编辑3次,待确认)

**编辑权限**:
- 仅广告所有者可编辑
- Active状态可编辑
- Expired状态可能需先Renew再编辑

**编辑后处理**:
- 编辑后广告可能移动到列表顶部(重新发布效果)
- 编辑不影响发布时间(原始发布时间保留)
- 编辑不影响浏览量/收藏量等统计数据

**确认页面**:
- 首次点击"Update my ad" → 显示确认页面
- 确认页面可能推荐推广套餐:"Want to move your ad back to the top"
- 再次点击"Update my ad" → 最终提交

### 3.4 删除操作规则

**删除确认**:
- 必须二次确认:点击"Delete ad" → 弹窗确认 → 选择"Yes" → 点击"Delete"按钮
- 确认对话框包含警告文案(如"This action cannot be undone")

**删除后处理**:
- 广告从Active ads列表移除
- 数据库软删除(推测,非物理删除)
- 照片保留在CDN(推测,7天后清理)
- 用户历史记录保留(可能在"Deleted ads"标签查看,待确认)

**删除限制**:
- 推广期内的广告:可能需先取消推广再删除 OR 直接删除且不退款(待确认)
- 有未读消息的广告:可能提示"You have unread messages"(待确认)

### 3.5 推广操作规则

**推广触发**:
- **新广告发布后**: 成功页面推荐推广套餐(Make your Ad stand out)
- **已发布广告**: Manage Ads页面点击"Promote"按钮

**推广套餐选择**:
- Featured: 3天(£0.50)/7天(£1.00)/14天(£1.80),默认7天
- Urgent: 7天(£0.75),固定天数
- Spotlight: 7天(£1.99),固定天数(可能需解锁条件)
- 允许多选:总费用=各套餐价格之和
- "Select all"快捷全选功能

**支付流程**:
- 支付页面可能出现隐私弹窗:"We Care About Your Privacy" → 必须点击"Accept all"关闭
- 点击"Pay Now" → 跳转3D Secure验证(可能)
- 3D Secure OTP验证:
  - 等待iframe加载(10秒)
  - 输入OTP:"1234"(测试环境固定值)
  - 点击"SUBMIT"提交
  - 验证成功后自动跳转
- 支付成功 → 跳转感谢页面(URL包含"thankyou")
- 超时处理:30秒内未完成支付 → 返回推广选择页

**推广生效**:
- 支付成功后立即生效(推测1-5分钟内)
- Featured: 广告置顶到分类列表最前面
- Urgent: 广告标题前显示"Urgent"标签
- Spotlight: 广告展示在Gumtree首页
- 推广期结束:自动恢复普通状态,不自动续费(除非用户勾选自动续费)

**自动续费**:
- 首次选择推广时可能弹窗提示:"Please confirm"/"automatically renew"
- 用户需点击"Confirm"确认
- 若勾选自动续费:推广期结束前自动扣款延期
- 可在账户设置中取消自动续费(待确认)

## 4. 错误处理

| 错误码 | 错误信息 | 触发条件 | 用户提示 | 处理方式 |
|-------|---------|---------|---------|---------|
| ERR_AD_NOT_FOUND | Ad not found | 广告已被删除/不存在 | "Ad not found or has been removed." | 刷新列表,移除该广告卡片 |
| ERR_EDIT_PERMISSION | No permission to edit | 非广告所有者尝试编辑 | "You don't have permission to edit this ad." | 跳转回列表页 |
| ERR_DELETE_FAILED | Delete failed | 后端API返回错误 | "Failed to delete ad. Please try again." | Toast提示,保留广告,可重试 |
| ERR_PROMOTE_FAILED | Promotion failed | 推广API调用失败 | "Failed to promote ad. Please try again." | 返回推广选择页 |
| ERR_PAYMENT_FAILED | Payment failed | 支付超时/失败 | "Payment failed. Please try again or choose a different payment method." | 返回支付页 |
| ERR_OTP_INVALID | OTP verification failed | 3D Secure验证失败 | "Verification failed. Please check your OTP code." | 重新输入OTP,最多3次 |
| ERR_LOAD_ADS_FAILED | Failed to load ads | 网络错误/API超时 | "Failed to load ads. Please refresh the page." | 显示"Refresh"按钮 |
| ERR_SESSION_EXPIRED | Session expired | 操作时Session过期 | "Your session has expired. Please log in again." | 跳转登录页,登录后返回 |
