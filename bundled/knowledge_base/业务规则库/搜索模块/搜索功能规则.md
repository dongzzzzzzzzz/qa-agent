# 搜索模块 - 搜索功能业务规则

## 1. 功能概述
- **功能描述**: Gumtree UK 站内搜索功能，支持关键词搜索、联想词、历史记录、筛选排序、地域过滤、保存搜索 alert、发布类目联想等能力
- **用户角色**:
  - 游客（未登录）：可执行搜索、筛选、排序、地域过滤
  - 注册用户（已登录）：额外支持保存搜索 alert、Post an ad 类目联想
- **入口位置**:
  - 首页顶部搜索框（placeholder: "Search Gumtree"）
  - 搜索结果页顶部搜索框
  - 首页分类导航栏（For Sale 等分类快捷入口）
  - Post an ad 发布向导（类目联想入口，`/postad/category`）
- **依赖模块**: 用户认证模块（保存搜索/发布类目需登录）、广告详情模块、3PA广告模块

## 2. 核心流程

### 2.1 主流程
1. 用户在首页顶部搜索框输入关键词
2. 系统实时展示联想词（输入 ≥2 个字符时触发，最多 10 条）
3. 用户点击搜索按钮 / 按 Enter / 选择联想词 触发搜索
4. 页面跳转至搜索结果页（SRP），展示匹配广告列表
5. 用户可在结果页进一步排序、筛选、地域过滤
6. 用户点击广告卡片跳转至广告详情页
7. 登录用户可保存搜索 alert

### 2.2 异常流程
- 无结果时：展示空状态页 + 三条建议文字 + Save search alert 按钮
- 拼写错误时：结果页 H1 显示 `0 ads for [词]`，并出现拼写纠正建议按钮（`Search instead for`）
- 特殊字符 / 纯空格：作为通配符处理，返回全部广告，不报错
- 空搜索框搜索：URL 无 `q` 参数，返回全部广告
- 局部搜索结果 < 100 条：额外展示 Nearby 扩展区域

## 3. 业务规则

### 3.1 输入规则
| 字段 | 类型 | 必填 | 长度 | 默认值 | 说明 |
|-----|------|-----|------|--------|------|
| 搜索关键词 (q) | String | 否 | 0-50 | - | maxLength=50，超出自动截断；空值时返回全部广告 |
| 搜索位置 (search_location) | String | 否 | - | United Kingdom | 支持城市名（London）、完整邮编（TW91EL）、Outcode（TW9）、United Kingdom |
| 最低价格 (min_price) | Number | 否 | - | - | 价格筛选下限（正整数） |
| 最高价格 (max_price) | Number | 否 | - | - | 价格筛选上限（正整数） |
| 排序方式 (sort) | Enum | 否 | - | 无（相关性） | 可选值：date / price_lowest_first / price_highest_first / distance |
| 分页 (page) | Number | 否 | - | 1 | 翻页参数，每页约 25 条 |
| 距离 (distance) | Number | 否 | - | 0.0001 | 配合 search_location 使用；0.0001 代表 Choose distance（全英范围） |

### 3.2 联想词规则
- 输入 **≥2 个字符**时触发联想词下拉，**单个字符不触发**
- 联想词列表最多显示 **10 条**
- 每条联想词中，输入的关键词以**粗体**高亮显示
- 每条含分类标注，格式：`[词] - [分类名]`（如 `dog - All Categories`、`dog - Dogs`、`dog - Pets for Sale`）
- 拼写错误时联想词显示模糊匹配结果，**不显示**原错误词本身
- 拼写纠正建议**仅在搜索后**的结果页出现，输入阶段不出现
- 关联搜索词标签（chips）显示在结果页标题下方，点击可跳转相关搜索

### 3.3 历史记录规则
- 用户点击搜索框（不输入任何内容）时展示 Recent Searches 历史记录下拉
- 点击历史记录项触发搜索，URL 参数：`search_term_populated_by=history`
- `Clear All` 按钮一键清除全部历史记录，下拉消失，搜索框失去焦点，页面不跳转
- 清除后再次点击搜索框，不再显示任何历史记录

### 3.4 URL 参数规则

| 触发场景 | search_term_populated_by 值 |
|---------|--------------------------|
| 直接输入并搜索 | `input` |
| 选择联想词搜索 | `suggested`（同时带 `searchOptionsExactMatch=true`）|
| 选择历史记录搜索 | `history` |
| 未修改参数触发（如纯空格） | `unchanged` |

通用默认参数：
- `keyword_correction=suggest`
- `search_location=United%20Kingdom`（或 `United+Kingdom`）
- `search_category=all`

### 3.5 排序规则
| 排序选项 | URL sort 参数值 | 说明 |
|---------|---------------|------|
| Most relevant first（默认） | 无 sort 参数 | 默认排序 |
| Most recent first | `date` | 按发布时间降序 |
| Price: Low to High | `price_lowest_first` | 价格升序；Free 视为 £0 参与比较 |
| Price: High to Low | `price_highest_first` | 价格降序；Free 视为 £0 参与比较 |
| Nearest first | `distance` | 按距离由近到远排列 |

⚠️ **Featured 置顶广告**（结果列表前 5 条）不遵循排序规则，排序验证需从第 6 条起

### 3.6 筛选规则
- **价格筛选**：左侧 Price 区域输入 min/max，点击 Search 生效；URL 参数：`min_price`、`max_price`
- **Make 筛选（车辆）**：选择车辆品牌后 URL 带 `vehicle_make=[品牌]`，同时自动限定 `search_category=cars`
- **Delivery Available**：toggle 开关，URL 参数：`support_shipping=true`；结果列表中所有广告卡片显示 `Delivery available` 标志
- **地理位置筛选**：左侧 Location 文本框输入位置 + Choose distance 下拉选择距离范围，点击 `Update` 生效；URL 参数：`search_location`、`distance`
- 恢复全英：将 Location 改回 `United Kingdom`，distance 恢复为 Choose distance，URL 中 distance 变为 `0.0001`

### 3.7 地域规则
- 支持输入格式：
  - 城市名：如 `London`（URL：`search_location=London`）
  - 完整邮编：如 `TW91EL`（URL：`search_location=TW91EL`）
  - Outcode：如 `TW9`（URL：`search_location=TW9`）
  - 全英：`United Kingdom`（URL：`search_location=United%20Kingdom`）
- 默认位置：`United Kingdom`（全英范围）

### 3.8 Nearby 扩展结果规则
- 局部搜索（非全英）结果数量 **< 100 条**时，自动展示 Nearby 扩展区域
- 扩展区域分隔标题：`Results from outside your search`
- 副标题：`Here are XXX additional results in your extended area`（XXX 为动态数量）

### 3.9 保存搜索 Alert 规则
- **仅限登录用户**使用；未登录用户点击后引导登录
- 搜索结果页和空状态页（无结果）均显示 `Save search alert` 按钮
- 保存成功弹出提示：`Your alert is set`
- 提示框包含 `Manage alerts` 链接
- Alert 管理页面 URL：`/my-account/saved-searches`
- 管理页面显示已保存 alert 数量和列表，支持删除操作（`Delete this saved search`）

### 3.10 发布类目联想规则（Post an ad）
- **仅限登录用户**；入口：首页 `Post an ad`（或 `Sell`）→ `/postad/category`
- 页面标识文案：`Tell us what category you are posting in`，下方为类目输入框
- 输入 **≥3 个字符**时触发 Suggested categories 列表，恰好显示 **5 条**
- 输入 **≤2 个字符**（如 `dr`）**不触发** Suggested categories
- 每条建议格式：`类目名`（路径用 ` > ` 分隔，带空格）+ `Select` 按钮
- 示例（输入 `dog`，zoidberg 实测）：
  1. Dogs（Pets > Pets for Sale）
  2. Petsitters & Dogwalkers（Services > Pets）
  3. Missing, Lost & Found（Pets）
  4. Vets（Services > Pets）
  5. Pet Toys（Pets > Equipment & Accessories）
- 页面始终显示 `Or browse to find a category` 链接/按钮
- 用例结束需登出，避免会话状态影响后续用例

### 3.11 分类导航规则（BRP）
- 首页搜索栏下方分类导航栏支持点击直达各分类列表页
- 鼠标悬停分类可展开子分类菜单，点击子分类跳转对应页面
- For Sale 列表页：URL `/for-sale`，Title `Used stuff for sale | Gumtree`，H1 格式 `XXX ads Stuff for Sale`
- Clothes 子分类页：URL `/for-sale/clothing`，Title `Used Clothes for Sale | Gumtree`，H1 格式 `XXX ads Used Clothes for Sale`

## 4. 错误处理
| 错误场景 | 触发条件 | 用户展示 |
|---------|---------|---------|
| 搜索无结果 | 关键词无匹配广告 | H1 显示 `0 ads for [词] in All Classifieds`；展示三条建议：`Check the spelling` / `Try adding or removing words` / `Try browsing the categories`；底部显示 `Save this search to receive email alerts...` + Save search alert 按钮 |
| 拼写错误 | 词无直接匹配、系统识别可能拼写错误 | 同无结果页面，额外在 H1 下方出现拼写纠正建议按钮，文案含 `Search instead for` |
| 特殊字符输入 | 搜索词含 `<>"'%*?` | 页面不报错，特殊字符作通配符处理，H1 显示 `ads All Classifieds` |
| 纯空格搜索 | 搜索框仅含空格 | 正常跳转，URL 中 q 为 URL 编码空格（`%20`），H1 显示 `ads All Classifieds` |
| 空搜索框直接搜索 | 搜索框为空 | 正常跳转，URL 无 q 参数，H1 显示 `ads All Classifieds` |
