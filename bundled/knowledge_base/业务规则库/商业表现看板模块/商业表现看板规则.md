# 商业表现看板模块 - 卖家广告表现分析看板业务规则

## 1. 功能概述
- **功能描述**: 商业表现看板(Performance Metrics Dashboard)为 Pro Account 卖家提供广告表现数据的可视化分析,包括搜索展示量、广告浏览量、回复量、地理位置分布等核心指标,帮助卖家了解商业收益表现并优化广告策略
- **用户角色**: Pro Account 卖家(仅Pro账号可访问)
- **入口位置**: 
  1. 左侧导航栏"My Metrics"链接(带"New"角标)
  2. 直接访问 `/performance-metrics` URL
- **依赖模块**: 
  - 用户认证模块(登录态验证)
  - 广告管理模块(Live Ads 数据源)
  - 消息模块(Unique Replies 数据源)
  - 数据导出模块

## 2. 核心流程

### 2.1 主流程
1. Pro用户从 `/manage/ads` 页面点击左侧导航栏"My Metrics"
2. 跳转至 `/performance-metrics` 页面
3. 页面默认加载"Last 30 days"时间范围的数据
4. 展示核心指标卡片区、Best Performing Locations、Ads Breakdown 表格
5. 用户可切换时间范围(Last 7/30 days),所有模块数据联动更新
6. 用户可点击 Export 按钮导出 Ads Breakdown 数据为 Excel 文件

### 2.2 异常流程
- **未登录用户访问**: 自动重定向至登录页,登录成功后返回 `/performance-metrics`
- **非Pro账号访问**: 显示提示"This account does not currently support the dashboard feature. Please upgrade to a Pro account."
- **无数据状态**: 各模块显示对应的空状态提示,Export 按钮禁用
- **网络请求失败**: 显示友好错误提示并提供重试机制
- **Session过期**: 收到401响应后跳转至登录页

## 3. 业务规则

### 3.1 权限规则
| 规则项 | 规则内容 |
|-------|---------|
| 访问权限 | 仅 Pro Account 用户可访问商业表现看板 |
| 导航显示 | 非Pro用户的侧边栏不显示"My Metrics"入口 |
| 直接访问拦截 | 非Pro用户直接访问 `/performance-metrics` 显示升级提示 |
| 登录态要求 | 未登录用户自动重定向至登录页 |
| 多账号切换 | UID关联多个账号时显示 Account Switcher,单账号时隐藏 |

### 3.2 时间筛选规则
| 字段 | 类型 | 必填 | 可选值 | 默认值 | 说明 |
|-----|------|-----|--------|--------|------|
| time_range | String | 是 | "last_7_days", "last_30_days" | "last_30_days" | 时间范围筛选 |

**筛选行为**：
- 默认加载时间范围为"Last 30 days"
- 切换时间范围后所有模块(核心指标、Locations、Ads Breakdown)数据联动更新
- 时间范围独立存储,刷新页面后保持用户上次选择
- Last 30 days 数据通常 ≥ Last 7 days 数据(30天包含7天)

### 3.3 核心指标计算规则

#### Search Views (搜索展示量)
- **定义**: 广告在搜索结果页/首页展示的总次数
- **数据源**: 搜索展示埋点统计
- **时间范围**: 基于所选时间范围统计

#### Ad Views (广告浏览量)
- **定义**: 广告详情页(VIP页)被查看的总次数
- **数据源**: 详情页浏览埋点统计
- **时间范围**: 基于所选时间范围统计

#### Unique Replies (去重回复量)
- **定义**: 去重后的买家回复数量
- **去重逻辑**: 同一 user + 同一 ad,3天内只计1次回复
- **数据源**: 消息模块回复记录
- **时间范围**: 基于所选时间范围统计

#### New Ads (新发布广告数)
- **定义**: 所选时间范围内新创建的广告数量
- **对应PRD**: "New Listings Posted"
- **数据源**: 广告创建记录

#### Reposted Ads (重发广告数)
- **定义**: 所选时间范围内重新发布的广告数量
- **对应PRD**: "Listings Reposted"
- **数据源**: 广告重发记录

#### Live Ads (在线广告数)
- **定义**: 当前 Active 状态的广告总数
- **特殊规则**: 不受时间筛选器影响,始终显示实时数据
- **展示位置**: 页面顶部欢迎语"You currently have X Live Ads."
- **数据一致性**: 必须与 Manage My Ads 页面的 Active 广告数一致

#### Search Views to Click Conversion (搜索转化率)
- **定义**: 从搜索展示到点击查看详情的转化率
- **计算公式**: (Ad Views / Search Views) × 100%
- **格式**: 百分比显示,保留至少1位小数(如"3.2%")
- **边界处理**: Search Views = 0 时显示"-"或"0%",不报除零错误

#### Ad Views to Reply Conversion (回复转化率)
- **定义**: 从查看详情到发起回复的转化率
- **计算公式**: (Unique Replies / Ad Views) × 100%
- **格式**: 百分比显示
- **边界处理**: Ad Views = 0 时显示"-"或"0%",不报除零错误

### 3.4 Best Performing Locations 规则

| 规则项 | 规则内容 |
|-------|---------|
| 数据来源 | 买家回复时的地理位置信息 |
| 地域范围 | 仅显示UK境内地区,不含海外 |
| 排序规则 | 按 Unique Replies 降序排列 |
| 展示列 | Location \| Unique Replies |
| 未知位置 | 位置未知的回复汇总显示为"Other" |
| 滚动支持 | 列表内可滚动查看所有地区 |
| 时间联动 | 随时间筛选器切换而更新 |

### 3.5 Ads Breakdown 表格规则

| 规则项 | 规则内容 |
|-------|---------|
| 展示列 | Listing \| Search Views \| Ad Views \| Unique Replies \| Location \| Features Used |
| 默认排序 | 按 Search Views 降序排列 |
| 数据范围 | 所选时间范围内任何时段 Live 的广告 |
| Listing显示 | 广告标题 + Listing ID,超长标题截断显示"...",悬停显示完整标题 |
| Location显示 | 广告的 L2 级地区(如"Birmingham") |
| Features Used | Bump Up 显示"+X次",Top Ad 显示"+X",未使用显示"None" |
| 数据缺失 | 单元格显示"—"(长破折号),不显示 null/undefined/空白 |
| 标题交互 | 广告标题不可点击(非链接) |
| 时间联动 | 随时间筛选器切换而更新 |

### 3.6 数据导出规则

| 字段 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| 导出格式 | .xlsx | 是 | Excel 文件格式 |
| 文件命名 | String | 是 | `live_ads_breakdown_last_7_days_YYYY-MM-DD` 或 `last_30_days`,日期使用T-1 |
| 导出列 | - | 是 | Listing Title, Ad ID, Search Views, Ad Views, Unique Replies, Location, Features Used |
| 数据一致性 | - | 是 | 导出数据必须与页面显示数据完全一致 |
| 按钮状态 | - | 是 | 有 Live Ads 时可用,无数据时禁用 |

### 3.7 空状态规则

| 模块 | 空状态提示文案 | 触发条件 |
|-----|-------------|---------|
| 指标卡片 | 显示"-"(连字符) | 所选时间范围内无数据 |
| Best Performing Locations | "No data available yet. Try polishing your ad to attract more replies." | 无回复数据 |
| Ads Breakdown | "No data available yet. Create a listing to unlock insights." | 无 Live Ads |
| 通用提示 | "We update the data everyday, but sometimes it may be delayed." | 页面底部固定展示 |

### 3.8 Tooltip 内容规则

| 指标 | Tooltip 内容 |
|-----|-------------|
| Search Views | "How many times your ad showed up when people were searching." |
| Search Views to Click Conversion | "The percentage of people who saw your ad in their search results and decided to click on it. It measures how effectively your photo, price, and title grab a user's attention." |
| Ad Views | "How many times your ad was opened to view the full details." |
| Ad Views to Reply Conversion | "The percentage of people who, after viewing your ad, decided to get in touch. This measures how well your photos, description, and price convinced the user to take the next step." |

### 3.9 数据一致性约束

| 约束项 | 约束内容 |
|-------|---------|
| Live Ads 一致性 | 商业表现看板页面的 Live Ads 数量必须与 Manage My Ads 页面的 Active 广告数一致 |
| Phase 1 KPI 一致性 | Manage My Ads 顶部 KPI 卡片数值(Last 7 days)必须与商业表现看板对应指标完全一致 |
| Unique Replies 汇总 | Ads Breakdown 中所有广告的 Unique Replies 总和应约等于账户级 Unique Replies |
| 时间范围逻辑 | Last 30 days 各指标值 ≥ Last 7 days 对应值 |
| 并发竞态 | 快速切换时间范围,最终数据必须与最后一次点击一致,不出现混合展示 |

### 3.10 埋点规则

| 事件名 | 触发时机 | 必需字段 |
|-------|---------|---------|
| dashboard_view | 页面首次加载完成 | user_id, timestamp |
| dashboard_time_range_change | 切换时间范围 | user_id, account_id, timestamp, time_range |
| dashboard_download_click | 点击 Export 按钮 | user_id, account_id, timestamp, time_range |
| dashboard_empty_state_view | 触发空状态展示 | user_id, account_id, timestamp, module_name, reason |

## 4. 错误处理

| 错误场景 | 错误信息 | 触发条件 | 用户提示 |
|---------|---------|---------|---------|
| 权限不足 | 403 Forbidden | 非Pro账号访问 | "This account does not currently support the dashboard feature. Please upgrade to a Pro account." |
| 未登录 | 401 Unauthorized | 无有效 session | 自动重定向至登录页 |
| Session过期 | 401 Unauthorized | Session 过期后触发 API 请求 | 自动跳转至登录页,登录后返回 |
| 网络异常 | 500/503 | API 请求失败 | 显示友好错误提示,提供重试机制 |
| 除零错误 | - | Search Views 或 Ad Views 为 0 时计算转化率 | 显示"-"或"0%",不报错 |
