# Buyer - 收藏广告业务规则

## 1. 功能概述
- **功能描述**：已登录用户可在首页/广告详情页收藏广告，并在 Favourites 列表中统一管理；未登录用户触发收藏操作时引导登录
- **用户角色**：已登录买家（完整功能）；未登录访客（触发登录拦截）
- **入口位置**：
  - 首页广告卡片爱心图标
  - 广告详情页顶部心形按钮
  - 底部导航栏「Saved」Tab → Favourites 列表
- **依赖模块**：登录业务域（未登录拦截）、首页业务域（广告卡片）

## 2. 核心流程

### 2.1 主流程

#### 2.1.1 收藏广告（首页）
1. 已登录用户在首页广告列表中点击广告卡片的爱心图标
2. 系统立即记录收藏，用户留在首页（不跳转）
3. 进入 Saved Tab → Favourites 页面，收藏的广告出现在列表中
4. 列表展示广告图片、标题、价格（£xx 或 FREE）

#### 2.1.2 取消收藏（Favourites 列表）
1. 进入 Favourites 页面
2. 点击广告卡片的爱心图标取消收藏
3. 广告从列表中消失（等待约 6 秒内生效）
4. 若为列表最后一条，显示空状态「Favourite items you like」

#### 2.1.3 取消收藏（广告详情页）
1. 从 Favourites 列表点击广告图片进入详情页
2. 详情页顶部心形按钮可见（显示「1 ❤」状态）
3. 点击心形按钮取消收藏
4. 返回 Favourites 列表，广告已消失（详情页操作同步生效）

### 2.2 异常流程
- **未登录访问 Saved Tab**：显示登录拦截页，包含「Log in」和「Sign up」两个按钮 → 点击「Log in」弹出登录方式选择弹窗（含 Continue with email）→ 邮箱登录成功后进入 Favourites
- **已登录时收藏列表为空**：显示空状态文案「Favourite items you like」
- **账号已保持登录态访问 Saved Tab**：跳过登录拦截，直接进入 Favourites 页面

## 3. 业务规则

### 3.1 输入规则
| 字段 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| 广告 ID | String | 是 | 被收藏广告的唯一标识 |
| 用户账号 | String | 是 | 收藏操作需登录态，邮箱格式 |
| 登录密码 | String | 是（登录时） | 测试账号 `1852134166@qq.com / Dong123!` |

### 3.2 校验规则
- 收藏操作前必须验证登录态；未登录则触发登录拦截页
- 登录拦截页必须同时展示「Log in」和「Sign up」两个按钮
- 邮箱登录成功判断：Favourites 页面「Favourites」标题可见，且「Search alerts」子 Tab 可见
- 取消收藏后广告从列表消失的等待时间：≤ 6 秒（首页收藏→列表）、≤ 8 秒（详情页取消→列表）
- 取消收藏数量校验：取消后剩余数量 = 取消前数量 - 1

### 3.3 权限规则
- **未登录访客**：点击 Saved Tab 触发登录拦截；无法执行收藏/取消收藏操作
- **已登录用户**：可收藏、查看收藏列表、取消收藏（首页卡片 & 详情页均支持）

### 3.4 业务约束
- 收藏操作完成后用户留在当前页面（首页），不发生跳转
- 广告详情页取消收藏操作必须同步反映到 Favourites 列表（实时同步，无需手动刷新）
- 广告列表卡片必须包含：图片、标题、价格（£xx 或 FREE）
- 空状态文案固定为：「Favourite items you like」
- 登录弹窗必须包含「Continue with email」选项
- 邮箱登录页必须包含：Email 输入框、Password 输入框、Continue 按钮、Forgot Password 链接

## 4. 错误处理
| 错误码 | 错误信息 | 触发条件 | 用户提示 |
|-------|---------|---------|---------|
| - | 登录拦截页 | 未登录点击 Saved Tab | 显示 Log in / Sign up 按钮 |
| - | 空状态 | 收藏列表为空 | 「Favourite items you like」 |
| - | 收藏不同步 | 详情页取消后列表仍显示 | ⚠️ 待确认（当前测试≤8秒同步） |

## 5. 已知问题
- **SAVE-002 标题模糊匹配**：验证收藏广告时使用标题前 20 字符模糊匹配，无法完全确认是同一广告（Jira 待追踪）
- **多平台差异**：iOS 和 Android 的登录拦截页 UI 元素可能存在细微差异，需分平台验证
- **登录态持久性**：自动化运行时若账号已登录，SAVE-001 的登录拦截步骤会被跳过，测试覆盖率降低
