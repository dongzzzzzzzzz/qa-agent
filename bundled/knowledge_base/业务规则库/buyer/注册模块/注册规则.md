# 注册模块 - 注册业务规则

## 1. 功能概述
- **功能描述**: Gumtree Unicorn 提供两套注册入口——首页弹窗注册（Sign up modal）与独立注册页（/create-account），均支持邮箱注册和社交注册，但在表单字段、校验逻辑、密码校验展示和营销同意方式上存在差异
- **用户角色**: 新访客（尚未拥有 Gumtree 账号）
- **入口位置**:
  - 弹窗入口：`www.unicorn.gumtree.io/` 顶栏「Sign up」按钮，或登录弹窗内「Don't have an account? Log in」切换
  - 独立页入口：`https://my.unicorn.gumtree.io/create-account`（直接访问，或独立登录页 REGISTER Tab 跳转）
- **依赖模块**: [登录模块](../登录模块/登录规则.md)（登录/注册弹窗共用 Modal 框架）、首页模块（触发入口）

## 2. 核心流程

### 2.1 主流程（注册弹窗）
1. 访客点击首页顶栏「Sign up」，弹出注册弹窗，标题「Sign up」
2. 选择注册方式（社交注册 或 Continue with email）
3. 邮箱注册：填写 First name、Last name、Email address、Password
4. 四项均合法后 Continue 按钮激活，点击提交
5. 注册成功 → 自动登录，跳转个人中心或首页（⚠️ 推断）
6. 邮箱已注册 → 显示「邮箱已存在」类错误（⚠️ 文案未实测）

### 2.2 主流程（独立注册页）
1. 访客访问 `/create-account`，查看左侧欢迎引导 + 右侧 REGISTER 表单
2. 选择注册方式（社交注册 或 邮箱表单）
3. 邮箱注册：Register 按钮始终可点，空提交后显示各字段行内错误
4. 所有字段合法 → 点击 Register → 注册成功自动登录（⚠️ 推断）
5. 已注册邮箱 → 显示「邮箱已存在」类错误（⚠️ 文案未实测）

### 2.3 异常流程
- 空表单提交（弹窗）→ Continue 按钮 disabled，无法提交
- 空表单提交（独立页）→ 各字段显示必填错误
- Email 格式非法 → 显示「Please enter a valid email address.」
- 密码过短（< 8 位）→ 显示「Too short. Please enter at least 8 characters.」
- 密码复杂度不足 → 显示复杂度要求错误文案
- 已注册邮箱 → 显示邮箱已存在错误（⚠️ 推断）
- 已登录用户访问 /create-account → 重定向至个人中心（⚠️ 推断）

## 3. 业务规则

### 3.1 输入规则

| 字段 | 类型 | 必填 | 格式 | 适用入口 | 说明 |
|-----|------|------|------|---------|------|
| First name | String | 是 | 非空 | 弹窗 + 独立页 | 用户名（名） |
| Last name | String | 是 | 非空 | 弹窗 + 独立页 | 用户名（姓） |
| Email address | String | 是 | 邮箱格式（含 @ 及域名） | 弹窗 + 独立页 | 需为未注册邮箱 |
| Password | String | 是 | 满足复杂度规则（见 3.2） | 弹窗 + 独立页 | 注册密码 |
| 营销邮件同意 | Boolean | 否 | Checkbox（弹窗）/ 默认同意（独立页） | 弹窗 + 独立页 | 可选 |

### 3.2 校验规则

**密码复杂度规则（5 条，弹窗实时展示 Checklist）**：
| 规则 | 说明 | 未满足时显示 | 满足时显示 |
|-----|------|------------|---------|
| One lower case letter | 至少一个小写字母 | – | ✅ |
| One upper case letter | 至少一个大写字母 | – | ✅ |
| One number | 至少一个数字 | – | ✅ |
| One special character | 至少一个特殊字符 | – | ✅ |
| Minimum 8 characters | 最少 8 个字符 | – | ✅ |

- 5条规则全部满足时，显示绿色文案「Strong password」（不再展示逐条 Checklist）
- 强密码示例：`Test1234!`

**弹窗校验逻辑**：
- 任一必填字段为空 → Continue 按钮保持 disabled（不可点击）
- Email 格式非法 → 字段红色边框 + 行内错误文案，Continue 保持 disabled
- 所有字段合法（含密码满足复杂度）→ Continue 激活（变绿色）
- 营销邮件 Checkbox 默认未勾选，勾选/取消不影响 Continue 激活状态

**独立页校验逻辑**：
- Register 按钮始终 enabled（空字段也可点击）
- 提交后显示各字段行内错误（红色文字 + 红色边框）
- 密码错误展示两行综合文案（过短 + 复杂度要求合并显示）

### 3.3 权限规则

| 角色 | 操作 | 行为 |
|-----|------|------|
| 未登录访客 | 访问 /create-account | 显示注册页 |
| 已登录用户 | 访问 /create-account | 重定向至个人中心（⚠️ 推断） |
| 已登录用户 | 查看首页顶栏 | 不显示「Sign up」按钮（⚠️ 推断） |

### 3.4 业务约束

**社交注册方式差异**：
| 社交方式 | 弹窗 | 独立页 |
|---------|------|------|
| Apple | ✅ Continue with Apple | ❌ 无 |
| Google | ✅ Continue with Google（iframe 内，新标签页） | ✅ Sign in with Google（iframe 内） |
| Facebook | ✅ Continue with Facebook | ✅ Sign in with Facebook（蓝色背景） |

**提交按钮差异**：
| 维度 | 弹窗 Continue | 独立页 Register |
|-----|--------------|---------------|
| 初始状态 | disabled（灰色） | enabled（绿色，始终可点） |
| 激活条件 | 所有字段合法 | 无条件（提交后显示错误） |

**密码校验展示差异**：
| 维度 | 弹窗 | 独立页 |
|-----|------|------|
| 触发时机 | 输入时实时展示 Checklist | 点击 Register 提交后 |
| 展示形式 | 5条规则逐条打勾 | 综合错误文案（两行） |
| 成功态 | 「Strong password」绿色文案 | 无特殊成功态展示 |

**营销邮件同意差异**：
| 维度 | 弹窗 | 独立页 |
|-----|------|------|
| 展示形式 | Checkbox 控件（可勾选/取消） | 纯文字说明 |
| 默认状态 | 未勾选 | 视为同意（默认） |
| 退订方式 | 不勾选即不同意 | 文字中含「this link」退订链接 |

**弹窗关闭与重置**：
- X 按钮关闭（实测 ✅），背景遮罩消失
- ESC 键关闭（⚠️ 推断）
- 关闭后重新打开 → 弹窗回到第一步（社交选项页），字段内容清空（⚠️ 推断）

**弹窗与登录切换**：
- 注册弹窗内点击「Already got an account? Log in」→ 弹窗切换为「Log in」登录表单（⚠️ 推断）
- 登录弹窗内点击「Don't have an account? Sign up」→ 弹窗切换为「Sign up」注册弹窗（实测确认 ✅）

**合规链接**：
- Terms of Use → `https://www.unicorn.gumtree.io/termsofuse`（弹窗 + 独立页均有）
- Privacy Notice → `https://www.unicorn.gumtree.io/privacy_policy`（弹窗 + 独立页均有）
- 独立页 Register 按钮下方也展示合规链接（实测 ✅）

**独立页页面结构**：
- 左侧欢迎区：「Welcome to Gumtree.」+ 5条好处列表（Send messages / Post ads / Rate users / Favourite ads / Set alerts）
- 右侧：LOGIN / REGISTER Tab 栏，当前激活 REGISTER

### 3.5 站点特殊规则

#### Unicorn 站（UK 测试站）
- 弹窗注册入口：`https://www.unicorn.gumtree.io/`（主站顶栏 Sign up）
- 独立注册页：`https://my.unicorn.gumtree.io/create-account`（my 子域）
- Terms of Use：`https://www.unicorn.gumtree.io/termsofuse`
- Privacy Notice：`https://www.unicorn.gumtree.io/privacy_policy`

## 4. 错误处理

| 错误场景 | 入口 | 触发条件 | 用户提示位置 | 提示文案 |
|---------|------|---------|------------|---------|
| 任一必填字段为空 | 弹窗 | 字段未填时 | Continue 按钮 disabled | 无文案 |
| 空 First name | 独立页 | 提交时字段为空 | First name 字段下方（红色） | 「Please enter your first name」 |
| 空 Last name | 独立页 | 提交时字段为空 | Last name 字段下方（红色） | 「Please enter your last name」 |
| 空/非法 Email | 弹窗 | 输入后 | Email 字段红色边框 + 下方文案 | 「Please enter a valid email address」 |
| 空/非法 Email | 独立页 | 提交时 | Email 字段下方（红色） | 「Please enter a valid email address.」 |
| 密码过短（< 8 位） | 独立页 | 提交时 | Password 字段下方（红色，第1行） | 「Too short. Please enter at least 8 characters.」 |
| 密码复杂度不足 | 独立页 | 提交时 | Password 字段下方（红色，第2行） | 「Include at least one capital letter, one lowercase letter, one number and one special character e.g. !@#$£%^*-_+=」 |
| 邮箱已注册 | 独立页 | 提交时服务端校验 | 待确认 | ⚠️ 未实测，文案推断 |
