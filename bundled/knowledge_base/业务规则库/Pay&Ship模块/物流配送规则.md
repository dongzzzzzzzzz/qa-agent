# Pay & Ship - 物流配送业务规则

## 1. 功能概述
- **功能描述**: 卖家创建物流面单后，系统通过第三方物流平台追踪包裹状态，从发货到签收的完整物流配送流程
- **用户角色**: 卖家（Seller）、买家（Buyer）、第三方物流公司
- **入口位置**: 卖家订单详情页 → Ship item → 创建面单页
- **依赖模块**: MangoPay 卖家注册模块、地址管理模块、第三方物流 API、Webhook 回调

## 2. 核心流程

### 2.1 主流程（正向物流）
1. 卖家在订单详情页点击"Ship item"
2. 系统检查卖家是否注册 MangoPay 卖家身份，未注册则跳转注册
3. 进入创建面单 preview 页面
4. 系统自动填充：发货地址（来自个人中心默认地址）、收货地址（来自订单地址快照）
5. 卖家选择投递点（Drop-off point），列表实时从物流平台拉取，按距离升序
6. 点击"Create label"，生成面单条码和追踪号
7. 卖家将包裹投递到选择的投递点
8. 物流状态通过 Webhook 回调：LABEL → AC → IT → DE（到家）/ SP（到自提点）→ DE（取件）

### 2.2 逆向物流（包裹退回）
1. 物流在运输中（IT/AT）无法投递：回调 RETURN 状态
2. 自提点超期未取件：自提商店 10 天超期 / 自提柜 3 天超期，回调 RETURN 状态
3. 订单状态变更为 Suspended（暂停）
4. 卖家签收退回包裹后，物流回调 DE 状态
5. 订单状态变更为 Returned to seller
6. 需客服介入处理退款

### 2.3 异常流程
- 卖家未注册 MangoPay 卖家身份：跳转 MangoPay 注册流程，完成后回到面单创建
- KYC 认证触发：跳转 KYC 认证页面，完成认证后继续
- 面单创建失败：提示错误信息
- 投递点列表加载失败：显示错误提示，建议重试

## 3. 业务规则

### 3.1 物流状态映射规则
| 物流状态码 | 物流状态描述 | 订单状态变更 | 触发动作 |
|-----------|------------|------------|---------|
| LABEL/NY | 卖家创建面单 | 无变化（仍为 Awaiting dispatch） | - |
| AC | 承运商接收包裹 | → On its way（在途中） | 更新 ship_date、estimated_delivery_date |
| IT | 运输中 | 无变化 | - |
| AT | 尝试配送 | 无变化 | 实际物流商很少调用 |
| SP | 送至自提点 | → Arrived（送达自提点）[仅自提点配送] / → Delivered（配送到家）[到家配送] | 更新 actual_delivery_date |
| DE | 已投递/签收 | → Delivered（已签收） | 更新 actual_delivery_date |
| RETURN | 退回 | → Suspended（暂停） | - |
| EX | 异常 | - | 待确认 |
| VOIDED | 面单作废 | - | - |
| INTERCEPTION | 物流已拦截 | - | - |

### 3.2 配送方式与 SP/DE 状态处理差异
| 配送方式 | SP 状态 | DE 状态 |
|---------|---------|---------|
| 配送到家 | 不适用 | 订单状态 → Delivered |
| 配送到自提点 | 订单状态 → Arrived | 买家取件后订单状态 → Delivered |

### 3.3 发货/收货地址规则
- 发货地址（Send From）：从卖家个人中心**默认地址**自动获取
- 收货地址（Send To）：从**订单地址快照表**自动获取（下单时的地址快照，后续修改不影响）

### 3.4 自提点超期规则
| 自提点类型 | 超期天数 | 超期后动作 |
|---------|---------|---------|
| 自提商店 | 10 天 | 自动退回卖家，物流回调 RETURN |
| 自提柜 | 3 天 | 自动退回卖家，物流回调 RETURN |

### 3.5 投递点列表规则
- Nearby 自提点：若有默认地址，使用该地址定位；若无，使用 London 默认定位
- Recently used（买家侧）：只有卖家成功创建面单的订单才会被记录，按使用频率倒序排列
- Nearby 最多展示 10 个自提点
- 搜索栏输入展示最多 20 个自提点
- 投递点列表从第三方物流平台**实时拉取**，按距离升序排列

### 3.6 权限规则
- 仅已登录的卖家账号可以创建面单
- 卖家必须先完成 MangoPay 卖家身份注册才能创建面单

### 3.7 业务约束
- 已创建面单后，买卖双方均不可取消订单
- 订单状态不回退（即使物流状态异常）
- 订单状态为 Delivered 后，不再关注物流状态变更
- 7 天内未发货（未创建面单），系统自动取消订单并退款

## 4. 错误处理
| 错误场景 | 用户提示 | 触发条件 |
|---------|---------|---------|
| 未注册 MangoPay 卖家身份 | 跳转注册页面 | 首次创建面单 |
| KYC 认证触发 | 跳转 KYC 认证 | 满足 KYC 触发条件 |
| 投递点加载失败 | 请刷新重试 | 物流 API 异常 |
