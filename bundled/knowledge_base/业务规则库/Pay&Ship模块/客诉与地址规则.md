# Pay & Ship - 客诉与地址管理业务规则

## 1. 功能概述
- **功能描述**: 买家在签收后上报商品问题（客诉），以及买卖双方管理配送地址的完整规则
- **用户角色**: 买家（Buyer）、客服（Customer Service）
- **入口位置**:
  - 客诉：买家订单详情页 → Report an issue
  - 地址管理：个人中心 → Addresses
- **依赖模块**: 订单模块、MangoPay 退款模块、Salesforce 客服系统、物流模块

## 2. 核心流程

### 2.1 客诉主流程
1. 买家进入订单详情页（订单状态：Delivered）
2. 点击"Report an issue"
3. 选择问题类型（商品不符 / 包裹未到达 / 运输损坏）
4. 根据问题类型决定是否上传图片
5. 填写问题描述，点击提交
6. 订单状态变更为 Issue reported
7. 系统向 Salesforce 发送工单，等待客服处理
8. 客服处理后：退款给买家 / 赔付给卖家 / 向物流申请赔付
9. 订单状态变更为 Closed

### 2.2 地址管理主流程
1. 买家进入个人中心 → Addresses
2. 添加/编辑/删除地址
3. 设置默认地址（用于 Checkout 自动填充和卖家发货地址获取）

### 2.3 异常流程
- 客诉 28 天后客服未处理：系统自动取消上报并确认收货（订单 → Completed）
- 买家取消上报（48小时内）：订单恢复 Delivered
- 买家取消上报（48小时后）：订单直接 → Completed

## 3. 业务规则

### 3.1 客诉问题类型与处理规则
| 问题类型 | 需要上传图片 | 客服处理方式 | 订单最终状态 |
|---------|------------|------------|------------|
| Item is not as described（商品描述不符） | ✅ 必须上传 | 全额退款买家，不对卖家 payout | Closed |
| Parcel didn't arrive（包裹未到达） | ❌ 不需要 | 退款买家 + 赔付卖家 + 向物流申请赔付 | Closed |
| Item is damaged in transit（运输途中损坏） | ✅ 必须上传 | 退款买家 + 赔付卖家 + 向物流申请赔付 | Closed |

### 3.2 客诉时间规则
| 时间节点 | 规则 |
|---------|------|
| 签收后 48 小时内 | 买家可上报问题、手动确认收货、取消上报（恢复 Delivered） |
| 签收超过 48 小时 | 系统自动确认收货；取消上报后订单直接 → Completed |
| 上报后 28 天未处理 | 系统自动取消上报并确认收货 → Completed |

### 3.3 上报 Salesforce 工单规则
- 问题上报成功后，系统自动向 Salesforce 创建客服工单
- 工单状态由客服系统独立管理
- 客服处理完成后，MangoPay 执行退款/赔付操作

### 3.4 地址管理规则
| 操作 | 规则 |
|------|------|
| 添加地址 | checkout 页面可操作；或在个人中心操作 |
| 修改/删除默认地址 | 仅在个人中心 Addresses 页面操作（checkout 不支持） |
| 设为默认 | 同时只能有一个默认地址；设新默认后原默认标识自动移除 |
| 删除默认地址 | 系统自动将第一个地址设为新默认 |

### 3.5 地址字段规则
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| Full name | String | 是 | 收件人姓名 |
| Address Line 1 | String | 是 | 街道地址 |
| Town/City | String | 是 | 城市 |
| Postcode | String | 是 | 邮政编码 |
| Phone | String | 是 | 联系电话 |
| Set as default | Boolean | 否 | 是否设为默认 |

### 3.6 校验规则
- 客诉图片：需要上传图片的问题类型必须上传至少 1 张图片，否则无法提交
- 地址所有字段均为必填

### 3.7 权限规则
- 仅买家可上报问题（卖家无此入口）
- 地址管理仅登录用户可操作
- 只能管理自己账号下的地址

### 3.8 评价规则
- 订单确认收货（Completed）后，买卖双方可互相评价
- 评价数据存储在 userreviews 库 feedback 表
- 订单评价与聊天评价共用同一入口逻辑：任一处评价后，另一处不再出现评价入口
- 评价提交后，订单详情页"Leave a review"按钮消失

## 4. 错误处理
| 错误场景 | 用户提示 | 触发条件 |
|---------|---------|---------|
| 未上传图片直接提交 | 请上传图片 | 问题类型需要图片但未上传 |
| 地址必填字段为空 | 请填写完整地址信息 | 提交地址时必填字段为空 |
