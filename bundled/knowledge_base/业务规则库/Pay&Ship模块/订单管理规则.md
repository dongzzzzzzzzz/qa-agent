# Pay & Ship - 订单管理业务规则

## 1. 功能概述
- **功能描述**: Pay & Ship 订单全生命周期管理，包含订单状态流转、取消规则、签收确认、买卖双方视角的订单列表与详情展示
- **用户角色**: 买家（Buyer）、卖家（Seller）
- **入口位置**: 顶部导航 → My Orders / 个人中心 → 订单列表
- **依赖模块**: 支付模块、物流模块、MangoPay 资金模块、评价模块

## 2. 核心流程

### 2.1 主流程（订单生命周期）
1. 支付成功 → Awaiting dispatch（待发货）
2. 卖家创建面单 → 仍为 Awaiting dispatch（订单状态无变化）
3. 承运商接收（AC 回调）→ On its way（在途中）
4. 配送到家：已投递（DE 回调）→ Delivered（已签收）
5. 配送到自提点：送达自提点（SP 回调）→ Arrived；买家取件（DE 回调）→ Delivered
6. 买家确认收货（手动或系统自动）→ Completed（已完成）
7. 确认收货 24 小时后 → 系统向卖家 payout

### 2.2 订单取消流程
1. 买家/卖家手动取消（仅限已支付未创建面单）→ Cancelled
2. 系统自动取消：5 分钟未支付 / 7 天未发货 → Cancelled
3. 取消后系统自动全额退款

### 2.3 异常流程
- 已创建面单后：买卖双方均不可取消，仅通过物流逆向/客诉处理
- 物流退回（RETURN）→ Suspended → 等待客服处理

## 3. 业务规则

### 3.1 订单状态枚举（买家视角）
| 状态 | 说明 | 对应 Tab |
|------|------|---------|
| Awaiting payment | 待支付（不暴露给用户） | - |
| Awaiting dispatch | 待发货（已支付/已创建面单） | Active |
| On its way | 在途中（AC 回调后） | Active |
| Arrived | 送达自提点（SP 回调后，仅自提点订单） | Active |
| Delivered | 已签收（DE 回调后） | Active |
| Issue reported | 问题已上报 | Active |
| Completed | 已完成（确认收货后） | Completed |
| Cancelled | 已取消 | Cancelled |
| Suspended | 暂停（包裹退回中） | Active |
| Closed | 已关闭（客诉处理完成） | Completed |
| Returned to seller | 已退回卖家 | - |

### 3.2 订单取消规则
| 取消场景 | 触发条件 | 操作方 | 退款规则 |
|---------|---------|--------|---------|
| 支付超时取消 | 5 分钟未完成支付 | 系统自动 | 无需退款（未支付） |
| 手动取消-待发货 | 已支付，未创建面单 | 买家/卖家 | 全额退款 |
| 系统自动取消-超时发货 | 已创建面单 7 天未实际发货 | 系统自动 | 全额退款 |
| 面单创建后 | 已创建面单 | 不可取消 | - |

**退款金额公式**：退款金额 = 订单总金额（商品费用 + 物流费 + 服务费）

### 3.3 签收与确认收货规则
| 场景 | 时间条件 | 操作/结果 |
|------|---------|---------|
| 买家手动确认收货 | 签收后 48 小时内 | 订单状态 → Completed |
| 买家上报问题 | 签收后 48 小时内 | 订单状态 → Issue reported |
| 买家取消上报 | 签收后 48 小时内取消上报 | 订单状态恢复 → Delivered |
| 买家取消上报 | 签收超过 48 小时后取消上报 | 订单状态直接 → Completed（不恢复 Delivered） |
| 系统自动确认收货 | 签收后 48 小时无操作 | 订单状态 → Completed |
| Payout 时机 | 确认收货后 24 小时 | 系统定时向卖家 payout |

### 3.4 操作按钮展示规则（按订单状态）
| 按钮 | 展示角色 | 展示条件 |
|------|---------|---------|
| Cancel order | 买家/卖家 | 待发货（未创建面单） |
| Ship item | 卖家 | 待发货 |
| Track shipment | 买家/卖家 | On its way / Arrived / Delivered |
| Confirm received | 买家 | Delivered 状态，48小时内 |
| Report an issue | 买家 | Delivered 状态，48小时内 |
| Cancel report | 买家 | Issue reported 状态 |
| Leave a review | 买家/卖家 | Completed 状态，未评价 |
| Contact seller/buyer | 买家/卖家 | Completed 状态 |

### 3.5 校验规则
- 取消原因"Something else"：需填写具体描述，不可为空，否则提示"Please provide details"
- 已创建面单后：取消操作按钮不展示

### 3.6 权限规则
- 非登录用户访问订单页：跳转登录页
- 访问他人订单详情：返回 403 错误或跳转自己的订单列表
- 买家不可执行卖家操作（如创建面单），反之亦然

### 3.7 业务约束
- 订单状态不回退（即使物流状态异常）
- 订单状态为 Delivered 后，系统不再处理物流状态变更
- 互相评价：订单评价与聊天评价结构相同，任一处评价后，另一处不再出现评价入口

## 4. 错误处理
| 错误场景 | 用户提示 | 触发条件 |
|---------|---------|---------|
| 未登录访问订单 | Please login to view your orders | 未登录直接访问订单 URL |
| 越权访问他人订单 | You don't have permission to view this order | 访问非自己的订单 |
| 取消原因为空 | Please provide details | 选择"Something else"但未填写描述 |
| 买家尝试卖家操作 | This action is only available for sellers | 跨角色操作 |
